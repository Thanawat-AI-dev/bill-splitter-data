# 🧾 หารบิล (Bill Splitter)

แอปสำหรับสร้าง "โปรเจกต์" คิดเงินแต่ละครั้ง กรอกเมนู/รายชื่อคน/ใครกินอะไร
แล้วสรุปเป็นแดชบอร์ด พร้อม Export เป็น PDF — ข้อมูลทุกโปรเจกต์จะถูกบันทึก
เป็นไฟล์ JSON ขึ้นรีโพ GitHub ของคุณเองโดยอัตโนมัติ ผ่าน GitHub REST API
(ไม่มีเซิร์ฟเวอร์ฝั่งหลังบ้าน รันเป็นเว็บไซต์ static ล้วน ๆ บน GitHub Pages ได้เลย)

แนวคิดของฟอร์มคำนวณอ้างอิงจากไฟล์ Google Sheet "Form_bill" (เมนู → รายชื่อคน →
ตารางนับ/หาร) แล้วนำมาทำเป็นเว็บแอปแบบ wizard 5 ขั้นตอน

## โครงสร้างไฟล์

```
index.html   หน้าเว็บหลัก
style.css    ดีไซน์ (โทนใบเสร็จ/Glassmorphism มืด-สว่าง)
app.js       ลอจิกทั้งหมด: wizard, การคำนวณ, GitHub API, กราฟ, Export PDF
firestore.rules  กฎความปลอดภัย Firebase Firestore สำหรับโหมดแชร์หลายคน
start-app.bat    เปิด local server สำหรับทดสอบบน Windows
```

## โหมดเก็บข้อมูลฟรีที่แนะนำ

### Firebase Firestore (แนะนำสำหรับแชร์ให้เพื่อน)
โหมดแชร์หลายคนใช้ Firebase Firestore + Anonymous Auth เพื่อไม่ต้องฝัง GitHub token ในลิงก์ และรองรับหลายคนกดพร้อมกันได้ดีกว่าการแก้ไฟล์ JSON บน GitHub

ขั้นตอน:
1. สร้าง Firebase project ด้วยแผน Spark (ฟรี)
2. เปิด Firestore Database
3. เปิด Authentication → Sign-in method → Anonymous
4. Project settings → Your apps → Web app → คัดลอก Firebase config
5. ในแอป กด ⚙️ แล้ววาง config ในช่อง Firebase
6. นำเนื้อหา `firestore.rules` ไปวางใน Firestore Rules แล้ว Publish

หลังจากตั้งค่า Firebase แล้ว ปุ่ม “แชร์ลิงก์” จะสร้างลิงก์ที่ไม่มี secret/token อยู่ใน URL

### GitHub mode
ยังใช้ได้สำหรับเก็บข้อมูลเป็น JSON ในรีโพส่วนตัว แต่โหมด guest share จะไม่ฝัง GitHub token ในลิงก์อีกแล้วเพื่อความปลอดภัย

## วิธีใช้งาน (ครั้งแรก)

### 1. สร้างรีโพ GitHub สำหรับเก็บข้อมูล
สร้างรีโพใหม่ (private ก็ได้) เช่น `bill-splitter-data` — แอปจะสร้างโฟลเดอร์
`projects/` และไฟล์ `projects/_index.json` ให้เองโดยอัตโนมัติตอนบันทึกครั้งแรก

> ถ้าต้องการให้แอปเองก็อยู่ในรีโพเดียวกับข้อมูลก็ได้ (ใช้รีโพเดียวสำหรับทั้ง
> โฮสต์เว็บผ่าน GitHub Pages และเก็บข้อมูล) หรือจะแยกรีโพข้อมูลออกไปอีกรีโพก็ได้
> เช่นกัน — ระบุ owner/repo ในหน้าตั้งค่าได้อย่างอิสระ

### 2. สร้าง Personal Access Token (PAT)

1. คลิกรูปโปรไฟล์มุมขวาบนของ GitHub → **Settings**
2. เลื่อนลงสุดเมนูซ้าย → คลิก **Developer settings**
3. คลิก **Personal access tokens** → เลือกแท็บ **Fine-grained tokens**
   (หรือเข้าตรงที่ `https://github.com/settings/personal-access-tokens/new`)
4. คลิก **Generate new token** แล้วกรอกฟอร์ม:
   - **Token name**: ชื่อให้จำง่าย เช่น `bill-splitter-app`
   - **Expiration**: เลือกอายุโทเค็น (สูงสุด 366 วัน — fine-grained ตั้ง "ไม่มีวัน
     หมดอายุ" ไม่ได้) แนะนำ 90 วัน แล้วค่อยสร้างใหม่ทีหลัง
   - **Resource owner**: เลือกบัญชีของคุณเอง (ถ้าเลือกเป็น Organization อาจต้อง
     รอแอดมินองค์กรอนุมัติก่อนใช้งานได้จริง)
   - **Repository access**: เลือก **"Only select repositories"** แล้วเลือกรีโพ
     ที่สร้างไว้เก็บข้อมูล (อย่าเลือก "All repositories")
5. เลื่อนลงมาที่ **Repository permissions** → หาแถว **Contents** → เปลี่ยน
   dropdown เป็น **"Read and write"** (ที่เหลือปล่อย "No access" ได้เลย แอปนี้
   ใช้แค่สิทธิ์นี้)
6. กด **Generate token** ที่ด้านล่างสุด
7. หน้าจอจะโชว์โทเค็นขึ้นต้นด้วย `github_pat_...` — กดคัดลอก **ทันที**
   เพราะ GitHub จะไม่แสดงค่านี้ให้ดูอีกเลยหลังจากออกจากหน้านี้
   (ถ้าหลุดดูไม่ทัน ต้องสร้างโทเค็นใหม่)

> ถ้ารีโพอยู่ใต้ Organization ที่ตั้งนโยบาย "Require administrator approval"
> โทเค็นจะ "รอการอนุมัติ" จนกว่า org owner จะกด Approve ก่อนใช้งานได้จริง

### 3. เปิดแอปแล้วเชื่อมต่อ
1. เปิดไฟล์ `index.html` (หรือลิงก์ GitHub Pages หลังดีพลอย)
2. กดปุ่ม ⚙️ ที่หัวเว็บ
3. กรอก Token, Owner (ชื่อบัญชี/องค์กร), ชื่อรีโพ, Branch (ปกติคือ `main`)
4. กด "บันทึกและทดสอบการเชื่อมต่อ"

โทเค็นจะถูกเก็บไว้ใน **localStorage ของเบราว์เซอร์นี้เท่านั้น** ไม่ถูกส่งไปที่อื่น
และไม่ถูก commit ขึ้น GitHub ไม่ว่ากรณีใด

### 4. เริ่มสร้างโปรเจกต์
กด "+ สร้างโปรเจกต์ใหม่" แล้วทำตามขั้นตอน:
1. **ข้อมูลบิล** — ชื่อ, วันที่, ร้าน, ส่วนลดรวม, ค่าบริการ %, VAT %
2. **เมนู** — รายการอาหารและราคา
3. **รายชื่อคน** — ใส่ชื่อทุกคนที่ร่วมบิล
4. **หารบิล** — ติ๊กว่าใครกินเมนูไหน (ระบบหารเฉลี่ยเฉพาะคนที่ติ๊ก)
5. **สรุปผล** — แดชบอร์ด + ใบเสร็จสรุป + ปุ่ม Export PDF

แต่ละขั้นตอนจะบันทึกขึ้น GitHub ให้อัตโนมัติเมื่อกด "ถัดไป" หรือ "ย้อนกลับ"

## วิธีคำนวณ

- ส่วนลดรวม (บาท) จะถูกหารเฉลี่ยให้แต่ละเมนูตาม **สัดส่วนราคา** ของเมนูนั้น
  เทียบกับยอดรวมทั้งบิล (ตามแนวคิดคอลัมน์ "สัดส่วนลด" ในไฟล์ Form_bill)
- ราคาหลังหักส่วนลดของแต่ละเมนู จะถูกหารเฉลี่ยเท่า ๆ กันเฉพาะคนที่ติ๊กว่ากิน
  เมนูนั้น (ตามแนวคิด "ตารางนับหาร")
- ค่าบริการและ VAT คำนวณจากยอดรวมหลังหักส่วนลด แล้วเฉลี่ยคืนให้แต่ละคน
  ตามสัดส่วนยอดที่ตัวเองกิน

## ดีพลอยขึ้น GitHub Pages

1. Push ไฟล์ทั้งหมด (`index.html`, `style.css`, `app.js`) ขึ้นรีโพ (จะใช้รีโพ
   เดียวกับที่เก็บข้อมูล หรือคนละรีโพก็ได้)
2. ไปที่ **Settings → Pages** ของรีโพนั้น → Source เลือก branch ที่ต้องการ
   (เช่น `main`) และโฟลเดอร์ `/ (root)`
3. รอสักครู่แล้วเปิดลิงก์ที่ GitHub มอบให้ (รูปแบบ
   `https://<username>.github.io/<repo>/`)

## ฟีเจอร์เพิ่มเติม (อัปเดตล่าสุด)

### 📷 อ่านเมนูจากรูปบิลอัตโนมัติ (ผ่าน Claude Skill)
แอปไม่ได้เรียก AI ในตัวเอง (ไม่ต้องใส่ API Key เพิ่ม) แต่ใช้วิธีนี้แทน:
1. ติดตั้งไฟล์ `bill-receipt-reader.skill` ที่แนบมาด้วย ผ่าน Claude.ai →
   **โปรไฟล์ขวาบน → Settings → Customize → Skills → "+" → "+ Create skill" →
   Upload** แล้วเลือกไฟล์ `.skill` นี้
2. เปิดแชทใหม่กับ Claude แล้วอัปโหลดรูปถ่ายบิล/ใบเสร็จ พร้อมพิมพ์ "อ่านเมนูจากรูปนี้"
3. Claude จะส่งไฟล์ `.json` กลับมาให้ดาวน์โหลด
4. ในแอป ไปที่ขั้นตอน **"2. เมนู"** กดปุ่ม **"📥 นำเข้าจากไฟล์"** แล้วเลือกไฟล์นั้น
   รายการจะถูกเติมเข้าตารางเมนูให้อัตโนมัติ (ควรตรวจสอบราคาอีกครั้งก่อนไปขั้นต่อไป
   เผื่อ Claude อ่านตัวเลขบางจุดผิดจากรูปที่เอียง/เลือนราง)

### 👥 รายชื่อคนที่บันทึกไว้ใช้ซ้ำ
ในขั้นตอน "3. รายชื่อคน" ทุกชื่อที่กรอกจะถูกบันทึกไว้ในกลุ่มเดียวที่ใช้ร่วมกันได้ทุก
โปรเจกต์ (เก็บไว้ที่ `people/_saved.json` บนรีโพ GitHub เดียวกัน) โปรเจกต์ถัดไปจะมี
ชิป "รายชื่อที่บันทึกไว้" ให้คลิกเพิ่มได้ทันทีโดยไม่ต้องพิมพ์ใหม่

### ✓ เลือกทุกคนต่อรายการเมนู
ในตารางขั้นตอน "4. หารบิล" มีคอลัมน์ "ทุกคน" เพิ่มมา — ติ๊กช่องนี้ในแถวเมนูไหน
จะติ๊กให้ทุกคนหารเมนูนั้นทันที (ใช้ปุ่ม "✓ ทุกคนหารทุกเมนูเท่ากัน" เดิมได้เหมือนกัน
ถ้าต้องการให้ทุกคนหารทุกเมนูทั้งบิลพร้อมกัน)

### 🧾 PDF ขนาดใบเสร็จจริง
ปุ่ม Export PDF ในหน้าสรุปผล จะได้ไฟล์ขนาดกว้างคงที่ 80mm (เท่าใบเสร็จร้านสะดวกซื้อ)
ยาวเท่าที่เนื้อหาต้องการ อยู่ในไฟล์เดียวหน้าเดียว ไม่ตัดแบ่งหน้าแบบ A4

---



- ใช้ **Fine-grained PAT** ที่จำกัดสิทธิ์เฉพาะรีโพที่ใช้เก็บข้อมูล และให้สิทธิ์
  เฉพาะ Contents (Read/Write) เท่านั้น
- อย่าแชร์ลิงก์หน้าจอที่มีโทเค็นปรากฏอยู่ และอย่า commit ไฟล์ settings ใด ๆ
  ที่มีโทเค็นอยู่ในนั้น (แอปนี้ไม่ทำเช่นนั้นอยู่แล้ว — โทเค็นอยู่ใน localStorage
  ของเบราว์เซอร์ผู้ใช้แต่ละคนเท่านั้น)
- หากใช้เครื่องคอมพิวเตอร์ส่วนรวม ให้กด "ล้างการเชื่อมต่อ" ในหน้าตั้งค่าก่อนเลิกใช้งาน
